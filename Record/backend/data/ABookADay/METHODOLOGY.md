# A Book a Day — 写稿方法论

> 写一本书的「一日精炼稿」之前，**先读这份文件**。
> 这是「怎么写」。新增书籍的字段规范在最下方「文件规则」。
> 本文件与 `BookInDepth/METHODOLOGY.md` 同源，**角色与铁律一致**，区别只在体量与媒介。

---

## ★★★ 第 0 条铁律：你的角色（不可妥协）

**你是「顶尖书籍提炼专家」。**

你不是博主、不是讲书人、不是励志作者、不是个人成长教练。

你做的事情只有一件——

**客观、准确地复述一本书的精华——以最短的篇幅、最高的思想密度，让用户在一次通勤的时间里读懂这本书。**

ABookADay 的定位是：让用户**快速读更多的书**。所以稿子要短，但「短」≠「水」，「短」=「极致取舍 + 极致密度」。

### 1. 必须做的

- **复述原书的核心论点链条**——前提 → 推论 → 结论，**不能跳步**。短篇幅意味着每一步都要更精炼，不是省略。
- **保留作者的原始声音**——他强调的、他重复的、他在意的——你都要展现。
- **大量复述书中的关键命题、定义、案例、实验**——读者读完应该感觉「这就是这本书在说什么」。
- **客观陈述事实**——作者写了什么、用什么例子论证什么——保持中立的复述者姿态。
- **极致取舍**——10000 字版本可以讲 12 个观点，4000 字稿要砍到 6–8 个**最核心**的。砍掉的不是次要的，是「为这本书做最短自我介绍时不会先讲的」。

### 2. 不要做的

- ❌ 过多的「读后感」「我个人的应用」「我打算落地的小动作」——这是 ABookADay 早期稿子的最大问题。
- ❌ 「愿你...」「希望你...」「让我们一起...」式励志收尾。
- ❌ 「行动指南」「7 步法」「实践清单」——除非书本身就是这种实用书。
- ❌ 替作者建立他没建立的跨书联想。
- ❌ 编造作者没说的话。
- ❌ 把「我」放进稿子里（除非是诚实的局限说明）。

### 3. 比例感（与 BookInDepth 一致，比例不因字数缩水而改变）

- **~80%** 是**对原书内容的客观复述**——论点、论据、推理、例子、定义、概念。
- **~15%** 是**必要的背景**——作者生平、写作背景、时代位置（**为理解原书内容服务**）。
- **~5%** 是**诚实的局限说明**——哪些被批评、哪些过时、哪些不严密。

### 4. 衡量标准

读者听完 / 读完你的稿子，应该能——

- 清楚说出**这本书在论证什么**
- 复述**作者的核心论点链**（哪怕只剩 6–8 个观点）
- 知道**作者用什么证据 / 例子论证**
- 理解**作者的核心概念定义**

**如果读者只记得你的"感悟"和"行动建议"——你失败了。**

## 为什么 ABookADay 也需要这套方法论？

体量小不等于标准低。**4000 字稿子的失败方式比 10000 字更隐蔽**——因为短，所以更容易塞进「我的感悟」和「励志收尾」充数；因为短，所以更容易省略论证链;因为短，所以更容易变成「博主讲书」而不是「书的客观提炼」。

ABookADay 与 BookInDepth 的真正区别：

| 维度 | ABookADay | BookInDepth |
|---|---|---|
| 字数 | ≥ 4000 中文字符（建议 4000–5500） | ≥ 10000 中文字符 |
| 媒介 | **听 + 读** 兼顾，TTS 友好 | **主要用来读** |
| 取舍 | 砍到 6–8 个最核心论点 | 完整展开 8–15 个 |
| 背景 | 浓缩到一段 | 单独一节展开 |
| 接受史 / 批评 | 一句话提及 | 单独一节展开 |
| 客观复述的姿态 | **完全一致** | **完全一致** |
| 第 0 条铁律 | **完全适用** | **完全适用** |

字数小的不是「降低标准」，是「更狠地取舍」。

## 五阶段流程

### 阶段 1：知识激活（Knowledge Excavation）

动笔之前，把这本书相关的一切先 dump 出来。**和 BookInDepth 一样要挖透**——不要因为最终只写 4000 多字就少挖。挖得越透，越知道该砍什么。

**至少覆盖 7 个维度**（比 BookInDepth 的 9 个少 2 个——「接收史」「最尖锐的批评」可以更轻）：

1. **作者**：生平、动机、时代位置（**1-2 句话能讲完的部分**）
2. **结构**：全书几部分，每部分讲什么
3. **核心论点**：用 1 句话能不能讲清楚？讲不清楚说明还没消化
4. **关键概念**：每一个核心概念给一个精确定义（用作者自己的话）
5. **响应了什么**：这本书在「回应」什么前人 / 现象（**简略**）
6. **横向连接**：和哪些其他书 / 思想家有内在亲缘（**只在作者本人引用了才用**）
7. **最大的局限**：诚实地列出（**1-2 条即可**）

**自检**：至少写满 20 个具体的、可入稿的信息点。少于 20 个 = 还不够厚 = 不要往下走。

---

### 阶段 2：极致取舍（Ruthless Selection）★ ABookADay 的核心阶段

这是 4000 多字稿子和 10000 字稿子最大的区别——**取舍**。

**问 3 个问题**：

1. **这本书的 1 句话命题是什么？** 写不出来 = 还没消化。
2. **如果只能讲 6–8 个观点，是哪 6–8 个？** 排序原则：(a) 不讲这条就不能理解这本书的命题；(b) 这条是作者最反复强调的；(c) 这条对当代读者最有解释力。
3. **每个观点必须搭配什么例子 / 实验 / 案例？** 优先用作者书中的例子，**不要替作者编例子**。

**砍的标准**：
- 砍掉「锦上添花」的，留下「必不可少」的
- 砍掉次级推论，留下主干论证
- 砍掉「我以为读者会喜欢」的，留下「这本书必须有」的

**自检**：把砍下来的清单拿给「不知道这本书」的人看 6–8 条，能不能让他看出全书的核心命题？

---

### 阶段 3：结构搭建（Skeleton）

ABookADay 推荐结构（**不是死板模板，按书的形态调整**）：

```
开场（约 400–600 字）
- 用一个让读者立刻进入的具体场景 / 问题 / 反直觉事实切入
- ★ 但绝对不要写成「博主开场白」——直接进书
- 引出这本书在回应什么问题

作者与书的背景（约 300–500 字，浓缩）
- 作者是谁（与理解这本书相关的部分）
- 这本书写于什么时代背景
- 这本书在它的领域 / 思想史里的位置（一两句）

核心论点深度复述（约 2800–3800 字，占大头）
- 沿着阶段 2 选出的 6–8 个核心观点逐个展开
- 每个观点内部：
  - 作者的论点（精确陈述）
  - 关键概念的定义（用作者自己的话）
  - 作者用的例子 / 实验 / 案例
  - （如有空间）作者怎么从前提推出结论
- ★ 这是稿子的主体，「这本书在讲什么」全在这里

局限 / 诚实说明（约 150–300 字，可选但推荐）
- 这本书的最大批评是什么（一句话）
- 哪些数据 / 案例已过时
- 不要写成「为作者辩护」，也不要写成「我的批判」

收尾（约 250–400 字）
- 把整本书的论证收束成可记忆的一句命题
- ★ 严禁「愿你...」「希望你...」式励志收尾
- ★ 严禁「我打算落地的小动作」式个人化收尾
- 最后一句要有「回头看的力量」——把书的核心命题用一句话钉死
```

---

### 阶段 4：一次写完（Draft in One Pass）

按骨架顺写。**不要边写边数字**。

**写作铁律**：

- **不要边写边数字**——数字是结果，不是写作时该想的事
- **遇到「我应该说但没说」的感觉 → 现在说**
- **遇到「我重复了」的感觉 → 停下，删掉，转下一节**
- **遇到「我要不要加一段我的感悟」的感觉 → 不要加**
- **遇到「我要不要给读者一个行动建议」的感觉 → 不要给**

**TTS 友好性（ABookADay 特有）**：

- 句子不要过长——每句中文 ≤ 40 字最佳；超过 60 字考虑断句。
- 段落自然换行——TTS 用换行做停顿。
- 数字尽量写汉字（「九十年代」而不是「90 年代」），但常用阿拉伯数字（年份、百分比、金额）可以保留。
- 避免表格、复杂符号、Markdown 表格。
- 避免长串英文术语；首次出现的英文概念在汉译后用括号给原文，之后只用汉译。
- 不要用 emoji。

---

### 阶段 5：复审 + 调音（Coherence + Voice Pass）

写完一遍后，**全文读一遍**（最好出声读一遍——这是 TTS 媒介的特别要求），问 5 个问题：

1. **主线贯通吗？** 6–8 个论点之间有没有内在的逻辑顺序？随便换序会不会更好？
2. **客观复述比例够吗？** 数一下「我」「我们」「你」开头的句子——这些是个人色彩浓的句子，应该 < 整稿的 10%。
3. **有没有励志收尾或个人建议尾巴？** 有 = 删掉重写收尾。
4. **听一遍流畅吗？** 有没有读起来拗口的长句？有没有歧义？
5. **没读过这本书的人，读完能不能讲清这本书的命题？**

**关键反应**：
- 主线断了 → 重排序
- 个人色彩过浓 → 改写成客观复述
- 励志收尾 → 删掉，重写一个「钉命题」式收尾
- 拗口 → 拆句子
- 命题讲不清 → **回阶段 2**，可能取舍出错了

---

## 字数硬规则

- **完整朗读正文必须 ≥ 4000 中文字符**（系统硬校验）
- **理想区间：4000–5500 字**——这是 ABookADay 的「黄金体量」
- **上限：不超过 6500 字**——超过就该考虑这本书是不是应该放进 BookInDepth
- **唯一不要做的事**：从 3800 字硬撑到 4000 字。**删掉重写**，不要补。

字数不是 target，是结果。如果按方法论走完写出 3800 字 → 是阶段 1 挖得不够 / 阶段 2 砍得过狠 → 回去重做，不要在结尾补一段感悟凑数。

## ★ 反「补丁式写作」铁律（与 BookInDepth 完全一致）

**严格禁止的流程**：
1. 写一稿
2. 查字数不够 4000
3. 在结尾加一段「我个人的应用」
4. 加一段「我打算落地的小动作」
5. 加一段「愿你...」
6. 达标

这是 ABookADay 旧稿的典型病灶。**全删**。

**正确的流程**：
1. 阶段 1 挖透（至少 20 个信息点）
2. 阶段 2 砍到 6–8 个核心
3. 阶段 3 搭骨架
4. 阶段 4 一次写完，不看字数
5. 写完后查字数：
   - ≥ 4000 且 ≤ 6500 → 进入阶段 5 复审
   - < 4000 → **删掉重写。不要补「感悟」凑数。**
   - > 6500 → 考虑是否搬去 BookInDepth；或回阶段 2 再砍

**判断标准**：每一段，你都能回答「这段存在的理由是什么」。如果回答是「为了凑字数 / 给读者打鸡血 / 让稿子有温度」，删掉。

---

## 写完之后的 3 个自检问题

1. **完全没读过原书的人能复述全书核心命题吗？** 如果你的稿子让没读过的人也能讲清楚「这本书在论证什么」，那你的取舍就是对的。
2. **整篇稿子里，"我 / 我们 / 你应该 / 你要" 开头的句子占比 < 10% 吗？** 高于 10% = 个人色彩过浓 = 不是客观提炼。
3. **结尾有没有把书的命题钉死？** 不是「愿你...」，不是「行动起来...」，是「这本书在说 X」。

任何一个回答「否」 → 这稿没写完。

---

## 一句话哲学

> **短，不是水的借口；短，是取舍的考验。**
> 4000 多字的客观提炼比 10000 字更难——因为你必须知道砍什么。
> 写出「这就是这本书」，不要写出「这是我对这本书的看法」。

# 文件规则（新增书籍）

每本书放在独立目录 `<book-id>/book.json`。

## 必填字段

- `id` 必须与目录名一致，使用英文小写、数字和短横线
- `shelfId` 使用现有书架：`wantToRead`、`reading`、`read`、`collection`
- `title`、`author`、`originalTitle`、`year`、`language`、`tags`、`myTake` 要尽量准确
- `sections` 必须包含：`oneLiner`、`synopsis`、`points`、`mindmap`、`quotes`、`notes`、`narration`

## 各 section 写法

- `oneLiner`：一句话浓缩这本书的命题（不是「我对这本书的感受」）
- `synopsis`：固定三段——背景、主张、适合谁读
- `points`：写 7–12 条，每条「观点 + 实际意义」（**比 narration 可以多铺一些**，points 是清单视图）
- `mindmap`：缩进式 Markdown，至少 3 层
- `quotes`：若不能确认原文，写「可记句 / 非原文摘抄」，避免伪造引用
- `notes`：可以包含跨书对照、个人化应用、反思问题（**notes 是允许个人色彩的字段**——narration 必须客观，notes 可以主观）
- `narration`：本方法论的核心约束对象——见上文五阶段流程

## `narration` 硬规则

- **直接写正文**——不要在正文前外挂「朗读大纲」「内容结构」「写作思路」之类的元信息块。骨架是隐性的，应该让读者在读正文时自然感受到，而不是先拿出一张解剖图给他看。
- 全文 **≥ 4000 中文字符**；理想区间 4000–5500 字，上限 6500 字。
- 用 `## 一级标题` 划分章节即可——前端从 `##` 抽取目录。**好的章节标题 = 好的目录 = 隐性骨架的外显**。
- 不需要在最前面 `# 书名`——元信息已经在卡片里。可以直接从开场段进入。
- 一次性成稿——不允许写短后用「感悟尾巴」补。
- 校验不足 4000 → 删除重写，不要补。
- 适合 TTS：句子不过长，段落自然换行，避免表格和复杂符号。

## 校验清单

新增或修改后至少检查：

- `book.json` 是合法 JSON
- 目录名和 `id` 一致
- 所有 `sections` 字段都存在且非空
- `narration` 直接是正文（**不包含** 「朗读大纲」「内容结构」之类的元信息标题块）
- `narration` 至少有 5 个 `##` 章节标题作为隐性骨架的外显
- `narration` 中文字符 ≥ 4000
- 三个自检问题（见上文）全部通过
- 第 0 条铁律的「不要做的」清单一条都没犯
